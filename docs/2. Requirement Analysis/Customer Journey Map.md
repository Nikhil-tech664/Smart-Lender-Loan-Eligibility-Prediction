## Customer Journey Map Table

Map out the applicant's experience step-by-step when applying for a loan eligibility assessment using SmartLender AI.

| Phase of Journey | Stage 1: Discovery & Entry | Stage 2: Submission & Analysis | Stage 3: Instant Feedback |
| :--- | :--- | :--- | :--- |
| **Actions**<br>*What does the customer do?* | - Navigates to the SmartLender Web portal.<br>- Reads introductory eligibility info.<br>- Selects desired loan parameters. | - Enters personal, financial, and credit history details.<br>- Clicks "Check Eligibility" button.<br>- Waits for validation and model loading. | - Views clear success or rejection message.<br>- Reviews recommendations or factors (DTI, LTI).<br>- Restarts or alters inputs for recalculation. |
| **Touchpoint**<br>*What part of the service do they interact with?* | - Home Page Header / Banner.<br>- Interactive sliders and input text boxes. | - Glassmorphic Form Container.<br>- AJAX Loader / spinner animation. | - Status Result card.<br>- Underwriting feedback and alert prompts. |
| **Customer Thought**<br>*What is the customer thinking?* | - "Is this page secure and fast?"<br>- "How long will this analysis take?" | - "Did I input my co-borrower income correctly?"<br>- "How is my credit history evaluated?" | - "The decision makes sense because of my credit score."<br>- "I can easily adjust the loan amount to qualify!" |
| **Customer Feeling**<br>*What is the customer feeling?* | - Curious and hopeful. | - Patient but anxious for outcomes. | - Relieved and satisfied with transparency. |
| **Process Ownership**<br>*Who is in the lead on this?* | - UI/UX Front-end developer. | - Flask Backend API and ML Engine. | - Underwriting Rules Engine. |
| **Opportunities**<br>*How can we improve this stage?* | - Introduce quick-fill templates for applicants. | - Highlight error fields instantly using local JS. | - Offer direct bank contact links for qualified profiles. |
